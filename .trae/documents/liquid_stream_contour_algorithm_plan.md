## Summary

- 目标：针对 `image/` 下现有样图，规划一套适用于单股液流、相机基本固定、光照可能变化场景的液流轮廓提取算法。
- 预期输出：液流左轮廓、右轮廓、中心线，以及可选的宽度随高度变化曲线。

## Proposed Changes

### 1. 算法总体路线

采用四层结构，避免把问题简化成单一步骤阈值分割：

1. ROI 稳定定位层
2. 光照归一化层
3. 液流候选增强与边界搜索层
4. 轮廓拟合与质量评估层

该路线的核心思想是：先消除光照变化带来的全局亮度偏差，再利用液流“单股、细长、近竖直”的几何先验做边界跟踪，从而提高容错性。

### 2. 模块规划

建议后续实现时按以下文件和模块拆分。以下文件当前尚不存在，属于规划中的新增项：

- `opencv-pro.cpp`
  - 作为程序入口
  - 负责读取图像、调用算法流程、保存调试结果和最终结果

- `src/preprocess.cpp` / `include/preprocess.h`
  - 负责灰度化、去噪、CLAHE、背景估计与光照归一化
  - 目标是把强中心亮斑和暗背景转化为更平坦的局部对比问题

- `src/roi_detector.cpp` / `include/roi_detector.h`
  - 负责定位中心观察窗或固定 ROI
  - 对固定相机场景优先支持固定 ROI；同时保留自动检测圆窗/椭圆窗的回退方案

- `src/stream_detector.cpp` / `include/stream_detector.h`
  - 负责液流候选区增强、主方向估计、粗中心线定位
  - 输出液流的候选列范围和倾角

- `src/contour_tracker.cpp` / `include/contour_tracker.h`
  - 负责逐行搜索左右边界、连续性约束、异常点修复
  - 输出左右边界点集、中心线点集、宽度序列

- `src/curve_fit.cpp` / `include/curve_fit.h`
  - 负责对离散边界点做平滑拟合
  - 目标是输出稳定可测量的轮廓，而不是噪声抖动边界

- `src/debug_visualizer.cpp` / `include/debug_visualizer.h`
  - 负责导出中间结果图：ROI、归一化结果、边缘响应、边界跟踪结果、最终叠加图
  - 这是后续调参与验收的重要支撑

### 3. 详细算法方案

#### 3.1 ROI 定位

优先方案：

- 在固定相机前提下，先通过人工标定得到稳定 ROI
- 后续运行时只处理 ROI 范围

回退方案：

- 对整图灰度化后做高阈值分割
- 提取最大亮区域
- 拟合椭圆或外接矩形作为观察窗 ROI

原因：

- 当前样图中心区域稳定、外围无效区域多
- 提前截取 ROI 可以显著减少暗角、遮挡、外圈纹理对后续边界检测的污染

#### 3.2 光照归一化

在 ROI 内进行以下处理：

1. 灰度化
2. 轻度去噪：`GaussianBlur` 或 `medianBlur`
3. 局部对比增强：`CLAHE`
4. 背景估计与扣除：
   - 方案 A：`GaussianBlur(大核)` 作为背景，再做减法
   - 方案 B：根据液流亮暗属性选择 `top-hat` 或 `black-hat`
5. 强度归一化到统一数值范围

规划建议：

- 将这一步设计成可切换参数的预处理管线
- 默认优先使用 `CLAHE + 大核背景扣除`

原因：

- 光照变化的本质是低频背景变化
- 液流边界和窄带结构属于高频信息
- 先做低频去除能显著提升不同光照下的一致性

#### 3.3 液流主方向与粗定位

目标：

- 找到液流大概在哪一列
- 估计它的倾角，为后续边界逐行搜索提供坐标系

建议做法：

1. 对归一化后的 ROI 计算 `Sobel X` 或 `Scharr X`
2. 统计每一列的边缘响应积分
3. 找到竖向响应最集中的列区域作为液流候选范围
4. 若检测到边界整体有轻微倾斜，可用以下任一方法估计角度：
   - Hough 直线
   - PCA 主方向
   - 候选边缘点的最小二乘直线
5. 倾角大于设定阈值时先做旋正

原因：

- 液流整体近竖直，使用定向梯度比直接阈值分割更稳
- 旋正后再逐行跟踪，连续性约束更容易施加

#### 3.4 左右边界逐行跟踪

这是主算法核心，建议作为主输出路径，而不是依赖 `findContours`。

处理流程：

1. 在候选列附近定义一条逐行扫描带
2. 对每一行提取横向灰度、梯度或组合响应曲线
3. 在每一行寻找一对边缘峰值，分别作为左边界和右边界
4. 下一行搜索时，只在上一行边界附近的小窗口中搜索
5. 加入以下约束：
   - 左右边界位移连续
   - 宽度变化平滑
   - 中心线变化平滑
   - 边界间距不能小于最小宽度或大于最大宽度
6. 当某几行边界响应弱或丢失时：
   - 允许短距离插值
   - 使用前后行趋势进行补点

推荐实现策略：

- 初版可用“逐行局部峰值搜索 + 平滑约束”
- 增强版可引入动态规划或 Viterbi，求解全局最优边界路径

原因：

- 用户需要的是左右轮廓，不只是区域分割
- 场景中高光、划痕、边缘暗角会干扰闭合轮廓检测
- 逐行跟踪能自然吸收几何先验和连续性约束

#### 3.5 曲线拟合与异常点处理

对原始边界点做后处理：

1. 使用中值滤波或滑动窗口剔除跳点
2. 使用 RANSAC 或残差阈值去除异常边界点
3. 使用三次样条或平滑样条拟合左、右边界
4. 中心线由左右边界中点计算得到
5. 宽度由同一行左右边界差值计算得到

输出结果：

- 左边界曲线
- 右边界曲线
- 中心线曲线
- 宽度统计
- 每帧质量评分

### 4. 容错性设计

为适应不同环境光照和成像扰动，规划中明确加入以下容错机制：

- 使用 ROI 限制处理区域，减少无关背景影响
- 使用局部对比增强和背景扣除，降低整体亮度变化影响
- 边界检测基于局部梯度和相对变化，而非固定灰度阈值
- 参数尽量采用自适应统计量生成，例如：
  - 边缘阈值根据 ROI 梯度分布动态确定
  - 允许搜索窗口依据当前液流宽度自适应扩展
- 对局部弱边缘使用连续性补偿，而不是直接判失败
- 为每一步输出质量指标，避免误检时静默输出错误轮廓

### 5. 备用方案与取舍

不建议作为主方案：

- 全局阈值分割
  - 对亮度变化、暗角和反光过于敏感

- 直接 `Canny + findContours`
  - 容易把观察窗边缘、划痕和反光一起提取出来

- 纯 Hough 直线拟合
  - 可以辅助估角，但不适合作为最终轮廓提取方法

可作为增强方向：

- 在传统算法稳定后，再尝试小型学习型分割模型
- 仅在后续场景出现明显姿态变化、多股液流或复杂背景时再考虑升级

## Assumptions & Decisions

### 关键假设

- 目标是单股液流
- 相机位置和视角基本固定
- 液流整体近竖直，仅有小角度倾斜
- 用户当前阶段需要算法规划，不需要立即编码实现

### 已锁定决策

- 主方法采用“光照归一化 + 几何先验边界跟踪”
- 输出定义为左右轮廓和中心线，不以二值掩膜为最终交付
- 初版以 OpenCV C++ 实现为目标，贴合当前工程
- 优先保证稳健性，其次再考虑实时性优化

   - 归一化不足
   - 边界搜索漂移

## 建议的实现顺序

1. 先做固定 ROI + 预处理 + 调试图输出
2. 再做液流粗定位和倾角估计
3. 再做左右边界逐行跟踪
4. 最后做样条拟合和质量评分
5. 完成后再做参数自适应与稳健性增强
